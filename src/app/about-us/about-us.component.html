This is a Dummy-Website created by me with an intention of testing my combinative and practical knowlege of Angular, HTML5, CSS3, TypeScript, Visual Studio Code, Java, Hibernate, SpringBoot and Eclipse.

What is the intention of this site?

Open-Library is a dummy website that allows users to exchange books among them.

How does this site work?

  • First you have to create an account by clicking CreateProfile where every feild in the form has to be filled.
  • Then u can click the SeeAllList to see your id.
  • Enter your correct id in the input feild and click Enter. You will be redirected into MyBooks.
  • Now u can add the book by clicking "Wanna add book?" where u have to fill the input feild of the form that will be displayed. (Note that image-link has to be added , not image.)
  • After adding books in your account, u can now click your books where u can update, delete or share with the other users if u want.
  • You can see the books you have shared by clicking My Shared Books where u can remove it whenever u want by first clickng the book and them by clicking Remove
  • You can click the notification inorder to see the request from other user who is interested in the book you have shared , where u communicate with them with the help of their email-id which will be shown in the notification pannel.
  • Likewise, u can also see the books shared by other users and can request to the owner if u found books on the list that u are interested on

Is this the best version of this site?

Definitely not. There are sitll many thing which can be improved on each step while using the site.Some of them are:

    • Proper utilization of toastr at appropriate events
    • Making the site responsive according to the size of browser
    • Accessing the user with username and password rather than automated generated ID
    • Adding cancelation button for the accidential clicking of certain buttons.
    • Allowing the users to communicate from within the site rather than using email.

 

    Still there are many improvements that can be done in both backend and frontend, which will be continuously improved.
